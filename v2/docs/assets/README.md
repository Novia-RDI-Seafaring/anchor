# Assets

Diagrams shipped with the v2 docs. Both vector-style line-art (Mermaid
exports, hand-drawn) and whiteboard-style raster art generated by Gemini
during a focused iteration round. Mixed origins are fine — we only care
that the final asset is accurate and the source path is traceable.

Older iterations (`-v1` through `-v16`, plus a draft pipeline + earlier
OIP) were pruned before the v2 public release to keep the repo clone
under 20 MB. Provenance lives in [`REVIEW.md`](./REVIEW.md) and the
Mermaid sources under [`source/`](./source).

## Naming policy

When iterating, always save to a new versioned filename (`-v18.png`,
`-v4.png`, etc.) rather than overwriting. This keeps a visible audit
trail without bloating the published repo — old iterations can stay in
your local working copy and be pruned before public commits.

## Source files

`source/architecture.mmd` and `source/style.md` accumulate the
structural ground truth and visual rules for the architecture diagram.
Use them as the spec for a designer redrawing the diagram in vector
form for a print poster.
