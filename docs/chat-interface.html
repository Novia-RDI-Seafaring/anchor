Chat Interface

Overview

The Chat Interface is your primary way to interact with Anchor UI. It provides a natural language interface to query your knowledge base and receive intelligent, context-aware responses.

Interface Components

Chat Panel

The chat panel appears on the right side of the workspace (or full-width on mobile):

  • Header: Shows "Assistant" title and close/minimize button
  • Chat Area: Displays conversation messages
  • Input Area: Text input with controls and file upload

Floating Chat Button

When the chat panel is closed, a floating button appears in the bottom-right corner. Click it to reopen the chat.

Starting a Conversation

New Chat

  1. Click "New Chat" in the sidebar, or
  2. Use the floating chat button if chat is closed
  3. Type your question in the input area
  4. Press Enter or click the send button (↑)

Empty State

When starting a new conversation, you'll see:

  • A welcome message: "How can I help?"
  • A brief description of capabilities
  • An empty message history

Message Display

User Messages

Your messages appear:

  • On the right side of the chat
  • With a user icon
  • In a dark background (light in dark mode)
  • With rounded corners (rounded-tr-sm)

Assistant Messages

Assistant responses appear:

  • On the left side of the chat
  • With a bot icon
  • In a light background with border
  • With rounded corners (rounded-tl-sm)
  • Streamed in real-time as they're generated

Input Area Features

Text Input

  • Auto-resizing: Expands up to 150px height as you type
  • Placeholder: "Type a message..."
  • Keyboard shortcuts:
    • Enter: Send message
    • Shift + Enter: New line

Context Controls

Toolbar above the input with toggle buttons:

  • Context On/Off: Toggle RAG (knowledge base) context
    • When ON (indigo): Queries use your knowledge base
    • When OFF (gray): General AI responses without document context
  • Web: Enable web search (coming soon)

File Upload

Upload documents directly from chat:

  1. When input is empty, click the paperclip icon (📎)
  2. Select files (PDF, DOCX, TXT, MD, HTML)
  3. Files upload and process automatically
  4. Status notifications show progress

Upload Status

File uploads show status indicators:

  • Uploading: Blue background with spinner
  • Success: Green background - "Added to KB"
  • Error: Red background with error message

Status notifications auto-dismiss after a few seconds.

How RAG Works in Chat

Query Process

When you send a message with Context enabled:

  1. Your question is sent to the backend agent
  2. The agent calls query_knowledge_base tool
  3. Your question is embedded using OpenAI
  4. Vector similarity search finds relevant chunks
  5. Top-K chunks are retrieved (default: 5)
  6. Chunks are passed to the LLM as context
  7. The LLM generates a response based on the context
  8. Response is streamed back to the chat

Active Document Filter

If you've selected a specific document in the header:

  • RAG queries are filtered to that document only
  • Only chunks from the selected document are retrieved
  • Useful for focused questions about specific documents

Conversation Management

Multiple Conversations

Anchor UI supports multiple concurrent conversations:

  • Each conversation has its own history
  • Conversations are listed in the sidebar
  • Click a conversation to switch to it
  • Messages persist across page reloads

Conversation Titles

  • Titles auto-generate from the first user message
  • Truncated to 30 characters if too long
  • Can be manually edited in the sidebar

Workspace Integration

Retrieved Context Display

The main workspace shows:

  • Source Documents: Cards showing which documents were used
  • Active Document: The formatted assistant response
  • Retrieved Context: Chunks with similarity scores

Synchronization

Chat and workspace are synchronized:

  • Each message triggers workspace updates
  • Retrieved chunks appear in the workspace
  • Source documents are highlighted

Best Practices

Effective Queries

  • Be specific: "What is the maximum operating temperature for component X?"
  • Use natural language: No special syntax needed
  • Ask follow-ups: Build on previous questions
  • Reference context: "According to the manual..."

Context Usage

  • Keep Context ON for document-based questions
  • Turn Context OFF for general AI questions
  • Select specific documents when asking about them

File Management

  • Upload files before asking questions about them
  • Wait for upload confirmation
  • Check processing status in Settings

Troubleshooting

Messages Not Sending

  • Check backend is running (port 8001)
  • Verify network connection
  • Check browser console for errors
  • Ensure input is not disabled

No Response from Assistant

  • Check backend logs for errors
  • Verify OpenAI API key is set
  • Check database connection
  • Ensure documents are processed

Context Not Working

  • Verify "Context On" is enabled
  • Check documents are uploaded and processed
  • Try broader search terms
  • Check similarity threshold settings
