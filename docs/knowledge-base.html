Knowledge Base Management

Overview

The Knowledge Base is the core of Anchor UI. It stores your documents, processes them into searchable chunks, and enables the RAG (Retrieval-Augmented Generation) functionality that powers intelligent responses.

Supported File Formats

Anchor UI supports the following document formats:

Format Extension Processing
PDF .pdf Full text extraction, tables preserved
Microsoft Word .docx Text and formatting extraction
Plain Text .txt Direct text processing
Markdown .md Markdown-aware processing
HTML .html HTML content extraction

Uploading Documents

Method 1: Chat Input Area

  1. Open the chat interface
  2. When the input is empty, click the paperclip icon (📎)
  3. Select one or more files from your computer
  4. Files will automatically upload and process
  5. You'll see upload status notifications

Method 2: Settings Page

  1. Navigate to Settings (gear icon in sidebar)
  2. Go to "Knowledge Base Management" section
  3. Click "CHOOSE FILES" button
  4. Select your documents
  5. Click "Upload" to process

Upload Process

When you upload a document, Anchor UI:

  1. Saves the file to the uploads directory
  2. Generates a document ID (MD5 hash of filename and content)
  3. Registers the document in the database
  4. Converts to markdown using Docling (for PDF/DOCX)
  5. Chunks the content into searchable segments
  6. Generates embeddings using OpenAI
  7. Stores in vector database for similarity search

Adding URLs

You can add web pages directly to your knowledge base:

  1. Go to Settings → Knowledge Base Management
  2. Find the "Add URL to Knowledge Base" section
  3. Enter a valid URL (e.g., https://example.com/article)
  4. Click "Add URL to Knowledge Base"
  5. The page content will be fetched and processed like a document

URL Processing

  • Content is fetched via HTTP/HTTPS
  • HTML is extracted and converted
  • Filename is derived from the URL path
  • Source URL is stored in metadata

Document Processing

Chunking Strategy

Documents are split into chunks for efficient search:

  • Chunk Size: Default 512 tokens (configurable)
  • Overlap: 50 tokens between chunks for context
  • Metadata: Each chunk stores position, document ID, and custom metadata

Embedding Generation

Each chunk is converted to a vector embedding:

  • Model: OpenAI text-embedding-3-large
  • Dimensions: 3072-dimensional vectors
  • Storage: PostgreSQL pgvector extension

Viewing Documents

Document List

View all documents in Settings → Knowledge Base Management:

  • Document filename
  • Number of chunks
  • Source type (file or URL)
  • Processing status

Document Statistics

The Knowledge Base shows:

  • Total Documents: Count of all uploaded documents
  • Processed Documents: Successfully processed count
  • Total Chunks: Sum of all chunks across documents
  • Status: Connection status to vector database

Selecting Active Document

Document Filter

Use the document selector in the header to focus queries:

  1. Click the document dropdown in the header
  2. Select "All Documents" or a specific document
  3. All subsequent RAG queries will be filtered to the selected document
  4. The filter persists until changed

Use Cases

  • Ask questions about a specific manual or guide
  • Compare information across documents
  • Focus on recent uploads
  • Isolate testing to one document

Managing Documents

Deleting Documents

  1. Go to Settings → Knowledge Base Management
  2. Find the document in the list
  3. Hover over the document row
  4. Click the delete icon (🗑️)
  5. Confirm deletion

Note: Deleting a document removes:

  • The document record from the database
  • All associated chunks and embeddings
  • The physical file from the uploads directory

Reingesting Documents

Re-process all documents in your knowledge base:

  1. Go to Settings → Knowledge Base Management
  2. Scroll to "Danger Zone"
  3. Click "Reingest documents"
  4. Confirm the action
  5. All documents will be re-processed (chunks regenerated)

When to Reingest:

  • After changing chunk size or overlap settings
  • If document processing seems incorrect
  • After updating embedding models
  • To refresh document metadata

Resetting Knowledge Base

⚠️ Warning: This action is permanent and cannot be undone!

  1. Go to Settings → Knowledge Base Management
  2. Scroll to "Danger Zone"
  3. Click "Reset"
  4. Confirm the action
  5. All documents and chunks will be deleted

Best Practices

Document Organization

  • Use descriptive filenames - they appear in search results
  • Upload related documents together
  • Keep documents updated - delete outdated versions
  • Organize by topic or project

Optimizing Search Quality

  • Upload well-structured documents (proper headings, sections)
  • Avoid very large single documents - split if possible
  • Ensure documents contain relevant, searchable content
  • Use consistent terminology across documents

Storage Management

  • Monitor chunk count - more chunks = more storage
  • Regularly review and remove unused documents
  • Consider document size vs. value
  • Archive old documents if needed

Troubleshooting

Upload Fails

  • Check file format is supported
  • Verify file is not corrupted
  • Ensure uploads directory is writable
  • Check file size limits

Processing Errors

  • Verify OpenAI API key is set
  • Check database connection
  • Review backend logs for errors
  • Try reingesting the document

No Search Results

  • Verify document was processed (check chunk count)
  • Ensure document filter isn't excluding results
  • Try broader search terms
  • Check similarity threshold settings
