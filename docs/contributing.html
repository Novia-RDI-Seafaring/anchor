Contributing to Anchor UI

Welcome Contributors!

Thank you for your interest in contributing to Anchor UI! This guide will help you get started with contributing to the project.

Getting Started

Prerequisites

  • Node.js 18+ and npm
  • Python 3.12+
  • PostgreSQL with pgvector
  • Git
  • Basic knowledge of React, TypeScript, Python, and FastAPI

Development Setup

  1. Fork the repository
  2. Clone your fork: 
    git clone https://github.com/your-username/anchor-ui.git
cd anchor-ui
  3. Install dependencies: 
    npm install
cd backend
uv sync
  4. Set up environment variables (see Getting Started)
  5. Start development servers: 
    npm run dev

Project Structure

Frontend (src/)

  • app/ - Next.js App Router pages and API routes
  • components/ - React components
  • contexts/ - React contexts for state
  • hooks/ - Custom React hooks
  • lib/ - Utility functions
  • types/ - TypeScript type definitions

Backend (backend/)

  • main.py - FastAPI application entry point
  • src/ - Source code modules
  • src/agent.py - PydanticAI agent
  • src/document_service.py - Document processing
  • src/vector_store.py - Vector database operations
  • src/tools/ - Agent tools

Development Workflow

1. Create a Branch

git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix

2. Make Changes

  • Write clean, readable code
  • Follow existing code style
  • Add comments for complex logic
  • Update documentation as needed

3. Test Your Changes

  • Test frontend changes in browser
  • Test backend APIs with curl or Postman
  • Verify no console errors
  • Check linting: npm run lint

4. Commit Your Changes

git add .
git commit -m "feat: add new feature description"
# or
git commit -m "fix: fix bug description"

Commit Message Convention:

  • feat: New feature
  • fix: Bug fix
  • docs: Documentation changes
  • style: Code style changes (formatting)
  • refactor: Code refactoring
  • test: Adding tests
  • chore: Maintenance tasks

5. Push and Create Pull Request

git push origin feature/your-feature-name

Then create a Pull Request on GitHub with:

  • Clear description of changes
  • Reference to related issues
  • Screenshots if UI changes

Code Style Guidelines

Frontend (TypeScript/React)

  • Use TypeScript for all new code
  • Follow React best practices
  • Use functional components and hooks
  • Memoize expensive components
  • Keep components small and focused
  • Use meaningful variable and function names

Backend (Python)

  • Follow PEP 8 style guide
  • Use type hints
  • Use async/await for I/O operations
  • Keep functions focused and small
  • Add docstrings to functions and classes
  • Use Pydantic for data validation

Pull Request Guidelines

Before Submitting

  • Ensure code follows style guidelines
  • Run linter and fix issues
  • Test your changes thoroughly
  • Update documentation if needed
  • Keep PRs focused and small

PR Description Template

## Description
Brief description of changes

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Breaking change
- [ ] Documentation update

## Testing
How was this tested?

## Checklist
- [ ] Code follows style guidelines
- [ ] Self-review completed
- [ ] Comments added for complex code
- [ ] Documentation updated
- [ ] No new warnings generated
- [ ] Tests added/updated

Communication

Getting Help

  • Check existing GitHub Issues
  • Search documentation
  • Ask questions in Issues
  • Be respectful and patient

Reporting Issues

When reporting bugs:

  • Use clear, descriptive titles
  • Provide steps to reproduce
  • Include error messages
  • Specify environment (OS, versions)
  • Add screenshots if relevant

Code Review Process

  1. PR is created and CI checks run
  2. Maintainers review the code
  3. Feedback is provided
  4. Changes are made if needed
  5. PR is approved and merged
