Settings

Overview

The Settings page provides comprehensive configuration options for your AI assistant and knowledge base. Access it by clicking the "Settings" button in the sidebar.

AI Assistant Configuration

System Prompt

The system prompt defines how your AI assistant behaves and responds:

  • Default: "You are a RAG-powered AI assistant with access to a knowledge base..."
  • Purpose: Instructs the agent to always query the knowledge base first
  • Customization: Edit the textarea to change behavior
Tip: The system prompt is critical for RAG functionality. The default prompt ensures the agent always uses your knowledge base before responding.

Model Selection

Choose the Large Language Model (LLM) for generating responses:

  • Available Models: Listed in the dropdown
  • Default: LlamaStack - ollama/llama3.2:3b
  • Model Detection: System automatically detects available models
  • Provider: Models can be from OpenAI, Azure OpenAI, or local (Ollama)

Advanced Settings

Temperature

  • Range: 0.0 to 1.0
  • Default: 0.7
  • Low (0.0-0.3): More focused, deterministic responses
  • Medium (0.4-0.7): Balanced creativity and consistency
  • High (0.8-1.0): More creative, varied responses

Max Tokens

  • Default: 2048
  • Purpose: Limits the maximum length of responses
  • Considerations:
    • Higher values = longer responses but more tokens used
    • Lower values = shorter, more concise responses
    • Balance between completeness and cost

MCP Tools

Model Context Protocol (MCP) tools extend agent capabilities:

  • Filesystem: File system operations
  • Web Search: Internet search capabilities
  • Database: Database query tools
  • Markitdown: Document conversion tools

See MCP Tools documentation for details.

Status Display

The status section shows current configuration:

  • System Prompt: Configured status
  • Selected Model: Current model name
  • Temperature: Current value
  • Max Tokens: Current limit
  • MCP Tools: Enabled/Disabled status
  • Knowledge Base: Document and chunk counts

Knowledge Base Management

Basic Info

Configure knowledge base metadata:

  • Name: "What are you working on?" - Knowledge base identifier
  • Description: "What are you trying to achieve?" - Purpose statement

Add URL

Import web content directly:

  1. Enter a valid URL in the input field
  2. Click "Add URL to Knowledge Base"
  3. Content is fetched and processed automatically

Management Operations

Refresh Status

  • Manually refresh document list and statistics
  • Updates counts and connection status
  • Useful after bulk operations

Upload Files

  1. Click "CHOOSE FILES" button
  2. Select one or more files (PDF, DOCX, TXT, MD, HTML)
  3. Click "Upload" to process
  4. Files are automatically chunked and embedded

Document List

View all documents in your knowledge base:

  • Filename and chunk count
  • Hover to reveal delete button
  • Scrollable list for many documents

Danger Zone

Warning: Operations in the Danger Zone are permanent and cannot be undone. Use with caution.

Reingest All Documents

Re-process all documents in the knowledge base:

  • What it does: Re-chunks and re-embeds all documents
  • When to use:
    • After changing chunk size or overlap settings
    • If processing seems incorrect
    • After updating embedding models
  • Time: Can take several minutes for large knowledge bases

Reset Knowledge Base

⚠️ This will DELETE ALL documents and chunks permanently!

  • What it deletes:
    • All document records
    • All chunks and embeddings
    • Physical files in uploads directory
  • When to use:
    • Starting fresh with new documents
    • Clearing test data
    • Resolving corruption issues
  • Confirmation: Requires explicit confirmation dialog

Configuration Best Practices

System Prompt

  • Keep instructions clear and specific
  • Emphasize knowledge base usage
  • Define response style and format
  • Test changes with sample queries

Temperature

  • Use lower values (0.3-0.5) for factual, technical content
  • Use higher values (0.7-0.9) for creative or exploratory tasks
  • Start with default (0.7) and adjust based on results

Max Tokens

  • 2048 is good for most use cases
  • Increase to 4096 for detailed explanations
  • Decrease to 1024 for concise answers
  • Consider API costs when increasing

Settings Persistence

Currently, settings are stored in the frontend state. For production use, consider:

  • Backend storage for system prompts
  • User preferences database
  • Configuration file support

Troubleshooting

Settings Not Saving

  • Check browser console for errors
  • Verify backend is running
  • Try refreshing the page

Model Not Available

  • Verify model is properly configured
  • Check API keys are set
  • Review backend logs

Knowledge Base Status Shows Error

  • Check database connection
  • Verify PostgreSQL is running
  • Review backend configuration
