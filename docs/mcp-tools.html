MCP Tools (Model Context Protocol)

What is MCP?

Model Context Protocol (MCP) is a standardized protocol that enables AI assistants to securely access external tools and data sources. It provides a way for AI models to interact with filesystems, databases, web services, and other resources through a consistent interface.

Key Concepts

  • Tools: Functions the agent can call to perform actions
  • Resources: Data sources the agent can access
  • Prompts: Templates for common interactions
  • Servers: MCP servers provide tools and resources

MCP in Anchor UI

Anchor UI supports MCP tools through the Settings page. You can enable or disable specific MCP servers to extend your agent's capabilities.

Available MCP Tools

1. Filesystem

  • Purpose: File system operations (read, write, list directories)
  • Endpoint: http://localhost:8000/sse
  • Use Cases:
    • Reading configuration files
    • Writing generated content to files
    • Listing directory contents
    • File manipulation tasks

2. Web Search

  • Purpose: Internet search capabilities
  • Endpoint: https://mcp.search-provider.com/sse
  • Use Cases:
    • Finding current information online
    • Researching topics not in knowledge base
    • Verifying facts
    • Getting latest updates

3. Database

  • Purpose: Database query operations
  • Endpoint: (Configurable)
  • Use Cases:
    • Querying structured data
    • Accessing external databases
    • Data analysis tasks
    • Integration with existing systems

4. Markitdown

  • Purpose: Document conversion and processing
  • Endpoint: http://localhost:8001/sse
  • Use Cases:
    • Converting documents to markdown
    • Processing various file formats
    • Extracting structured content

Enabling MCP Tools

Step-by-Step

  1. Navigate to Settings (gear icon in sidebar)
  2. Scroll to "MCP Tools" section
  3. Toggle individual tools on/off using the switches
  4. Tools are enabled/disabled immediately

Configuration

Each MCP tool requires:

  • Server Endpoint: URL where the MCP server is running
  • Authentication: May require API keys or tokens
  • Network Access: Backend must be able to reach the endpoint

How MCP Works

Architecture

The MCP integration follows this flow:

  1. User enables MCP tools in Settings
  2. Frontend sends configuration to backend
  3. Backend connects to MCP servers
  4. Agent receives available tools
  5. Agent can call tools during conversations
  6. Tool results are returned to the agent
  7. Agent incorporates results into response

Tool Execution

When the agent needs to use an MCP tool:

  1. Agent identifies need for external data/action
  2. Agent calls appropriate MCP tool
  3. MCP server executes the operation
  4. Results are returned to the agent
  5. Agent uses results to generate response

Security Considerations

Access Control

  • MCP tools have access to configured resources
  • Filesystem tools can read/write files
  • Web search may access external APIs
  • Database tools can query databases

Best Practices

  • Only enable tools you need
  • Use secure endpoints (HTTPS when possible)
  • Restrict filesystem access to safe directories
  • Monitor tool usage in logs
  • Use authentication for sensitive tools

Setting Up MCP Servers

Local MCP Servers

For local development, you may need to run MCP servers:

  1. Install MCP server packages
  2. Configure server endpoints
  3. Start servers on specified ports
  4. Update endpoint URLs in Settings

Remote MCP Servers

For production or shared servers:

  • Ensure servers are accessible from backend
  • Configure authentication if required
  • Use secure connections (HTTPS)
  • Monitor server health and availability

Troubleshooting

Tools Not Working

  • Verify MCP server is running
  • Check endpoint URL is correct
  • Review backend logs for connection errors
  • Test endpoint accessibility

Connection Errors

  • Check network connectivity
  • Verify firewall settings
  • Check SSL/TLS certificates
  • Review authentication credentials

Tool Execution Failures

  • Check tool permissions
  • Verify input parameters
  • Review server logs
  • Test tool manually if possible
