Component Overview

Frontend Components

Layout Components

Sidebar (src/components/layout/Sidebar.tsx)

  • Purpose: Navigation and conversation management
  • Features:
    • Conversation list with edit/delete
    • New chat button
    • Settings access
    • Dark mode toggle
    • Collapsible on mobile
  • State: Uses AppContext for conversations and settings

Header (src/components/layout/Header.tsx)

  • Purpose: Document selection and model configuration
  • Features:
    • Document dropdown selector
    • Model selector
    • RAG pipeline status
    • Document count display
  • API Calls: Fetches documents, sets active document

MainContent (src/components/layout/MainContent.tsx)

  • Purpose: Display workspace with artifacts and context
  • Features:
    • Source documents grid
    • Active document display
    • Retrieved context chunks
    • Similarity scores
  • Data Source: CopilotKit visibleMessages and tool results

Chat Components

ChatInterface (src/components/chat/ChatInterface.tsx)

  • Purpose: Main chat container
  • Features:
    • Minimizable panel
    • Message display
    • Input area integration
    • CopilotKit integration
  • Props: isOpen, onClose, initialMessages

ChatArea (src/components/chat/ChatArea.tsx)

  • Purpose: Display conversation messages
  • Features:
    • User and assistant message bubbles
    • Empty state with welcome message
    • Memoized for performance
    • Auto-scrolling
  • Props: messages, isEmpty

InputArea (src/components/chat/InputArea.tsx)

  • Purpose: Message input and file upload
  • Features:
    • Auto-resizing textarea
    • Context controls (RAG toggle, Web)
    • File upload with status
    • Keyboard shortcuts (Enter to send)
  • API Calls: Document upload endpoint

Settings Components

SettingsPage (src/components/settings/SettingsPage.tsx)

  • Purpose: Application configuration
  • Sections:
    • AI Assistant Configuration
    • Knowledge Base Management
  • Features:
    • System prompt editor
    • Model selection
    • Temperature and token controls
    • MCP tools configuration
    • Document upload/management
    • Danger zone operations

UI Components

AgComponents (src/components/ui/AgComponents.tsx)

  • Purpose: Reusable UI components from AG-UI
  • Components:
    • AgButton - Styled buttons
    • AgCard - Card containers
    • AgInput - Text inputs
    • AgSelect - Dropdown selectors
    • AgBadge - Status badges
    • AgToggle - Toggle switches

Context Components

AppContext (src/contexts/AppContext.tsx)

  • Purpose: Global application state
  • State:
    • Current view (workspace/settings)
    • Sidebar open/closed
    • Chat open/closed
    • Dark mode
    • Selected model
    • Active conversation
    • Conversations list

Hooks

useConversationHistory (src/hooks/useConversationHistory.ts)

  • Purpose: Manage conversation persistence
  • Features:
    • LocalStorage persistence
    • Create/update/delete conversations
    • Active conversation tracking

Backend Components

Core Services

DocumentService (src/document_service.py)

  • Purpose: Document orchestration
  • Methods:
    • upload_file() - Process file uploads
    • upload_url() - Process URL content
    • process_document() - Convert, chunk, embed
    • list_documents() - Get all documents
    • delete_document() - Remove document
    • search() - Vector similarity search
    • get_stats() - Knowledge base statistics

VectorStore (src/vector_store.py)

  • Purpose: Vector database operations
  • Methods:
    • add_document() - Register document
    • add_chunks() - Store chunks with embeddings
    • search() - Similarity search
    • list_documents() - Query documents
    • delete_document() - Remove document and chunks
    • reset() - Clear all data

EmbeddingsService (src/embeddings.py)

  • Purpose: Generate text embeddings
  • Model: OpenAI text-embedding-3-large
  • Methods:
    • embed_text() - Single text embedding
    • embed_texts() - Batch embeddings

Agent Components

Agent (src/agent.py)

  • Purpose: PydanticAI agent definition
  • Configuration:
    • Model selection
    • System prompt
    • State management
    • Tool registration

RAG Tools (src/tools/rag_tools.py)

  • query_knowledge_base: Vector similarity search
  • check_db_status: Database health check
  • add_to_conversation: Message history

Processing Components

DoclingConverter (src/docling_processing/docling_converter.py)

  • Purpose: Convert documents to markdown
  • Supported: PDF, DOCX
  • Features: Table preservation, fast mode

MarkdownFormatter (src/docling_processing/formatters.py)

  • Purpose: Chunk markdown content
  • Strategy: Configurable chunk size and overlap

API Components

FastAPI App (main.py)

  • Endpoints:
    • /agent - AG-UI agent endpoint
    • /api/documents/* - Document management
    • /api/search - Knowledge base search
    • /api/stats - Statistics
    • /api/active-document - Document filter
    • /health - Health check

Component Relationships

Frontend Flow

page.tsx (Root)
├── AppProvider (Context)
├── Sidebar
├── Header
├── MainContent
│   └── Uses CopilotKit messages
└── ChatInterface
    ├── ChatArea
    └── InputArea

Backend Flow

main.py (FastAPI)
├── /agent (AG-UI)
│   └── agent.py (PydanticAI)
│       └── tools/rag_tools.py
│           └── DocumentService
│               ├── DoclingConverter
│               ├── MarkdownFormatter
│               ├── EmbeddingsService
│               └── VectorStore
└── /api/* (REST)
    └── DocumentService

Data Flow Between Components

Chat Message

  1. InputArea → ChatInterface
  2. ChatInterface → CopilotKit
  3. CopilotKit → Next.js API
  4. Next.js API → FastAPI /agent
  5. Agent → query_knowledge_base tool
  6. Tool → DocumentService.search()
  7. DocumentService → VectorStore.search()
  8. Results flow back through layers
  9. MainContent displays results

Component Best Practices

Frontend

  • Use TypeScript for type safety
  • Memoize expensive components
  • Extract reusable logic to hooks
  • Keep components focused and small

Backend

  • Use async/await for I/O
  • Implement singleton pattern for services
  • Validate inputs with Pydantic
  • Handle errors gracefully
